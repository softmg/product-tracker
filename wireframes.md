### Поведение и UX-детали

```text
Навигация:
  - Дашборд — страница по умолчанию после входа в систему
  - Доступен по клику на логотип «Product Tracker» из любого экрана
  - Пункт «Дашборд» в левом сайд-баре (если он есть) или в верхней навигации

Блок «Ожидают моего действия»:
  - Подсказка в колонке «Действие» — конкретный следующий шаг, не просто статус
  - Если список пуст → «Нет задач. Всё в порядке ✓» (зелёный)
  - Максимум 5 строк, остальные скрыты под «Показать ещё N»

Блок SLA:
  - !! красный — уже просрочено
  - !  жёлтый — дедлайн в течение 2 дней
  - Сортировка: сначала просроченные, затем по дедлайну по возрастанию
  - Клик на строку → открывает карточку гипотезы на нужной вкладке

Лента изменений:
  - Только события по гипотезам, где пользователь — ответственный или участник
  - Фильтр: [Все события v] → можно выбрать тип: смена статуса / скоринг / артефакты / комментарии
  - Клик [→] → переход к конкретному объекту (карточка, артефакт, комментарий)

Ближайшие дедлайны:
  - Показывает задачи подэтапов Deep Dive и экспериментов из всех активных гипотез под моим управлением
  - Не только мои задачи, но и задачи команды по моим гипотезам (чтобы PM видел всю картину)
  - Просроченные дедлайны окрашены красным, завтрашние — жёлтым

Обновление данных:
  - Дашборд обновляется автоматически каждые 60 секунд или по клику [↻ Обновить]
  - Счётчик уведомлений [🔔 3] в шапке — живой, обновляется без перезагрузки страницы
```

### Поведение и UX-детали

```text
Счётчик в шапке [🔔 N]:
  - Показывает количество НЕПРОЧИТАННЫХ уведомлений
  - Обновляется в реальном времени (polling или WebSocket)
  - При N=0 иконка без цифры, без визуального акцента
  - При N>99 показывает «99+»

Dropdown-панель:
  - Открывается по клику на иконку, закрывается кликом вне панели или нажатием Esc
  - Показывает последние 5 уведомлений (только непрочитанные + 1–2 прочитанных для контекста)
  - Клик на уведомление → переход к объекту + отметка как прочитанное
  - «Открыть полный центр →» → переход на полную страницу 0б

Полная страница:
  - Группировка по датам (Сегодня / Вчера / 01.04 / ...)
  - Фильтр по типу: Смена статуса / Назначение / SLA / ПК / Артефакты / Комментарии
  - Фильтр по прочитанности: Все / Только непрочитанные
  - Фильтр по гипотезе: выпадающий поиск по ID/названию
  - Клик на строку → переход к объекту + автоматически отмечает как прочитанное
  - «Отметить все как прочитанные» — сбрасывает счётчик в шапке

Прочитанность:
  - ● непрочитанное: жирный текст, цветная левая полоса
  - ○ прочитанное: обычный текст, серая полоса или без полосы
  - Уведомление помечается прочитанным при клике на него или на [ Открыть → ]

Уведомления от «системы»:
  - SLA-алерты генерируются автоматически фоновым процессом
  - В поле «Инициатор» показывается «система», не пользователь
```

### Поведение карточки на доске

```text
┌─────────────────────────────────┐
│ HYP-002                         │   ← кликабелен, открывает карточку
│ Краткое название гипотезы       │
│ ─────────────────────────────── │
│ Иванов → Кузнецов               │   ← инициатор → ответственный
│ Скоринг: 8.4   [Средний]        │   ← бейдж приоритета
│ SLA: !  01.04 – 03.04           │   ← SLA-статус + даты
│                           [···] │   ← контекстное меню: открыть /
└─────────────────────────────────┘     изменить статус / назначить
```

### Правила переключения видов

```text
[ ☰ Таблица ] [ ⊞ Доска ]
     ↑               ↑
  активный       неактивный
(подчёркнут,    (серый, кликабелен)
 акцентный цвет)

- Выбранный вид сохраняется в localStorage (запоминается между сессиями)
- Фильтры и поиск работают одинаково в обоих видах
- В табличном виде фильтр [Статус v] присутствует
- В виде доски фильтр [Статус v] скрыт (статус = колонка)
- Пагинация в таблице; на доске — «+ Ещё N» в каждой колонке (infinite scroll)
```

```text
Новый:          Добавлен в список, контакт ещё не установлен
На связи:       Отправлено приглашение / идёт согласование даты
Запланирован:   Дата и формат встречи согласованы
Проведён:       Интервью состоялось, боли зафиксированы
Отказ:          Респондент отказался — причина фиксируется в заметках
```

### Поведение и UX-детали

```text
Доступ к CRM:
  - Открывается из строки подэтапа 5 «Поиск респондентов» в чек-листе Deep Dive
  - Также доступна из подэтапа 6 «Интервью»
  - Отдельного пункта меню нет — только через карточку гипотезы

Таблица респондентов:
  - Колонка «Боли» показывает число зафиксированных болей; клик → список тегов
  - Контекстное меню [···]: Открыть карточку / Изменить статус / Удалить
  - Строка с ❌ Отказ визуально приглушена (серый текст)
  - Строка с 📅 Запланирован подсвечивается, если дата сегодня или завтра

Карточка респондента:
  - Открывается как боковая панель (не уходим со страницы CRM)
  - Поле «Откуда» (источник контакта) — свободный текст: LinkedIn, реком., конф. и т.д.
  - Боли добавляются тегами из глобального словаря тегов гипотезы
  - Новый тег можно создать прямо из поля (free-tagging)

Сводка болей:
  - Доступна по кнопке «Сводка болей» над таблицей
  - Обновляется в реальном времени при добавлении новых интервью
  - Процент подтверждения боли учитывается в чек-листе Deep Dive автоматически:
    если боль упомянута ≥ 3 раза из 5 интервью → пункт «Подтверждённая боль» отмечается
```

## 21) Сводный UX-поток переходов и блокировок

```text
Идея
  └─(все обязательные поля заполнены?)─Да─> Первичный скоринг
                                        └Нет: блок + подсветка полей

Первичный скоринг
  └─(score >= threshold_primary?)──────Да─> Deep Dive
                                        └Нет: блок перехода

Deep Dive
  └─(обязательный checklist complete?)─Да─> Эксперимент
                                        └Нет: блок + список недостающих пунктов

Эксперимент
  └─(обязательные эксперименты/метрики?)─Да─> Питч
                                           └Нет: блок перехода

Питч
  └─(голосование завершено?)────────────Да─> Решение принято
                                        └Нет: блок фиксации решения

Решение принято
  ├─Go──────> (далее в реализацию вне MVP)
  ├─No-Go───> Архив
  └─Iterate─> Deep Dive / Эксперимент (по решению комитета)
```
